现在我要增加时间轮和心跳检测模块：首先需要Entry类跟TcpConnection挂钩，若Entry还在轮子里连接还存活；一旦被踢出轮子且没别的地方引用则析构关闭连接。构造函数传入TcpConnectionPtr，需要一个弱指针指向Tcpconnection。核心容器Bucket:其实 Bucket 不需要单独写个类，通常用类型别名定义即可。

组成：
using EntryPtr = std::shared_ptr<Entry>;

using Bucket = std::unordered_set<EntryPtr>;（或者 std::vector，但 unordered_set 语义更准）。

using Wheel = std::deque<Bucket>;（双端队列，方便头删尾插）。最后是管理类TimerWheel:成员变量：
Wheel wheel_: 存储 Bucket 的队列。设置60秒超时，只用设计单层时间轮即可。

size_t idleSeconds_: 超时阈值。

TcpServer* server_: 指向服务器的指针，用于获取 loop。

成员函数：
onTimerTick(): 被 TimerQueue 每秒调用一次。

逻辑：wheel_.push_back(Bucket());（插入一个新桶）

wheel_.pop_front();（弹出最旧的桶，这一步会触发里面所有 Entry 的析构）。

updateConnection(conn): 当连接有新消息时调用。

逻辑：从 conn 的 any（上下文）中取出 EntryPtr，重新 push_back 到 wheel_.back()。

模块名称,核心职责,关键细节
生命周期包装 (Entry),绑定连接与轮子的寿命,利用析构函数执行 shutdown
刻度容器 (Bucket),存放当前秒内活跃的所有连接,仅存储 shared_ptr<Entry>
轮子调度器 (TimingWheel),每秒推进刻度，清理过期连接,维护 deque<Bucket> 的长度
TcpConnection 适配,存储自身的 EntryPtr,使用 std::any (C++17) 存放上下文
